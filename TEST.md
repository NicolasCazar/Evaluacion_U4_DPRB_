# Evaluación de Conocimientos: GitHub y Markdown
 
## 1. Conceptos Generales
**¿Qué es un archivo Markdown y para qué sirve?**
> Un archivo Markdown es un tipo de archivo de texto que usa símbolos simples para dar formato, como títulos, listas o enlaces. Sirve para escribir información de forma ordenada y fácil de leer, especialmente en plataformas como GitHub.
 
**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**
*El README.md sirve como una presentación del proyecto. Ahí se explica de qué trata, cómo funciona, cómo usarlo y cualquier dato importante para que otras personas lo entiendan sin problema. Esto se pudo evidenciar y analizar facil en el video que nos muestra como realizar este proceso.*

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** (Es cuando guardo los cambios que hice, pero solo en mi computadora.)
* **Push:** (Es cuando envío esos cambios a GitHub para que se suban al repositorio.)
 
**¿Para qué sirve el botón "Fetch Origin"?**
> Sirve para ver si hay cambios nuevos en el repositorio sin descargarlos todavía, así puedo saber si alguien más hizo modificaciones.(Nicolas Cazar)
